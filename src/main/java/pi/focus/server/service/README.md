## Пакет `service/` — Бизнес-логика

**Назначение:** Подготовка данных. Преобразование внутренних структур в публичные контракты (`api/`).

### Структура:
*   `service/*/mocks/`: 
    *   Классы, реализующие интерфейсы сервисов.
    *   Возвращают захардкоженные тестовые данные.
    *   Аннотированы `@Profile("dev")` (активны только в режиме разработки).
    *   Позволяют фронтенду видеть "живой" сайт без подключенной базы данных.
*   `service/*`: 
    *   Будущие классы, работающие с реальными данными
    *   Активируются в профиле `prod`. 
#### Важное замечание:
Механизм профилей распространяется только на Context - классы. Модели (DTO) общие для всех профилей.

### Принцип работы:
Spring внедряет нужную реализацию сервиса в контроллер в зависимости от активного профиля (`dev` или `prod`). Код контроллера при этом не меняется.
